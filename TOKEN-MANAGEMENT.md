# 🪙 Token Management Guidelines

## วันที่สร้าง: 22 มี.ค. 2026

---

## 🎯 เป้าหมาย
- ประหยัด Token ให้มากขึ้น
- ตอบกระชับ ไม่ยืดยาว
- เตือนพี่ลภเมื่อใช้เยอะเกินไป

---

## 📋 กฎการใช้ Token

### ✅ ทำ:
1. **ตอบสั้นกระชับ** - ไม่ต้องอธิบายยืดยาว
2. **รวมงานทำทีเดียว** - ถ้ามีหลายอย่าง ทำพร้อมกัน
3. **ใช้ model เหมาะสม** - งานเล็กใช้ model เล็ก
4. **Heartbeat ใช้ประโยชน์** - ไม่ต้อง HEARTBEAT_OK ทุกครั้ง

### ❌ หลีกเลี่ยง:
1. ตอบซ้ำซ้อน
2. อธิบายมากเกินไป
3. ค้นหาข้อมูลที่ไม่จำเป็น
4. สร้างไฟล์ที่ไม่จำเป็น

---

## 🤖 การปรึกษาทีม

| งาน | ปรึกษา |
|-----|--------|
| เรื่องเงิน/ประหยัด | น้องขุนคลังแก้ว |
| เรื่องฉุกเฉิน/เล็กน้อย | น้องใจดี (Och) |
| เรื่องซับซ้อน/หลัก | น้องแก้วใส |

---

## ⚠️ เตือนเมื่อ:
- ถามต่อเนื่องเกิน 5 ครั้ง
- ข้อมูลยาวเกิน 2000 ตัวอักษร
- ใช้ model ใหญ่กับงานเล็ก
- ทำงานซ้ำซ้อน

---

## 📊 Model Selection

| Model | ใช้กับ |
|-------|--------|
| Claude 3 Haiku | งานเล็ก, ถาม-ตอบเร็ว |
| Claude 3.5 Sonnet | งานปกติ, coding |
| Claude 3 Opus | งานซับซ้อนมาก |

**Och (น้องใจดี)** ใช้ **Claude 3 Haiku** - ประหยัดที่สุด!
